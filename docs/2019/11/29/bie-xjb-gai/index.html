不要没事就发光发热

Posted on 2019-11-29 In review

最近有几件事做得不太满意。

一是在不了解业务上下文，没有测试保证的情况下，随意重构代码，反倒破坏了某个统计功能，导致数据库比真实数据偏小，PM被喷得有点惨。

二是心血来潮，到了一个新项目就忙着优化Spring Boot版本和Pipline，导致没法部署到集成环境。

以上故事告诉我们，不要没事就发光发热。

重构，要有充分的测试来保证。

“优化”的前提是“能work”。
